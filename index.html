🔍 Nota para Novatos: ¿Qué es una API de IA?

Imagina que quieres que tu aplicación pueda "ver" imágenes. En lugar de programar desde cero cómo reconocer objetos (lo cual tomaría años), usas una API de Computer Vision que ya sabe hacerlo.

Analogía simple: Es como usar Google Translate en tu app en lugar de crear tu propio traductor.

Conceptos básicos que debes dominar:

  • Endpoint: La dirección web donde está tu servicio de IA
  • API Key: Tu "contraseña" para usar el servicio
  • Request/Response: Envías datos (imagen, texto) y recibes resultados (análisis, traducción)

Decisiones Arquitectónicas Críticas

¿Cuándo usar Azure AI Services?

✅ USA cuando:

  • Necesitas funcionalidad estándar (OCR, traducción, análisis de sentimientos)
  • Quieres implementación rápida
  • No tienes datos específicos para entrenar
  • El presupuesto es limitado

❌ NO uses cuando:

  • Necesitas comportamiento muy específico de tu industria
  • Tienes grandes volúmenes de datos propios
  • Requieres control total del modelo

¿Cuándo usar Azure OpenAI?

✅ USA cuando:

  • Necesitas generación de contenido
  • Conversaciones complejas
  • Análisis de texto avanzado
  • Creatividad y razonamiento

❌ NO uses cuando:

  • Necesitas respuestas determinísticas
  • Datos altamente sensibles sin control
  • Presupuesto muy limitado

🔍 Nota para Novatos: ¿Qué es la identidad en la nube?

Analogía simple: Imagina tu oficina. Puedes entrar con:

  1. Llave física (API Key): Funciona, pero si la pierdes, cualquiera puede entrar
  2. Tarjeta de empleado (Azure AD): Más segura, se puede desactivar remotamente
  3. Reconocimiento facial (Managed Identity): La más segura, no puedes "perderla"

Responsible AI Principles

Los 6 Principios de Microsoft:

  1. Fairness: IA debe tratar a todos equitativamente
  2. Reliability & Safety: IA debe ser confiable y segura
  3. Privacy & Security: Proteger datos y privacidad
  4. Inclusiveness: IA debe empoderar a todos
  5. Transparency: IA debe ser comprensible
  6. Accountability: Humanos deben ser responsables

🔍 Nota para Novatos: ¿Por qué es importante la IA responsable?

Ejemplo real: Un sistema de IA para revisar CVs discriminaba contra mujeres porque fue entrenado con datos históricos sesgados.

Impacto:

  • Legal: Demandas por discriminación
  • Reputacional: Pérdida de confianza del público
  • Ético: Perpetuación de injusticias sociales

Tu responsabilidad como AI Engineer:

  • Entender los datos de entrenamiento
  • Probar con grupos diversos
  • Implementar human oversight
  • Documentar decisiones y limitaciones

🔍 Nota para Novatos: ¿Qué puede "ver" una computadora?

Analogía simple: Imagina que describes una foto a alguien por teléfono. Computer Vision hace exactamente eso, pero de forma automática y estructurada.

Conceptos básicos que debes dominar:

  • Bounding box: Rectángulo que rodea un objeto detectado
  • Confidence score: Qué tan "segura" está la IA (0.0 a 1.0)
  • Tags vs Categories: Tags son específicos ("perro"), categories son generales ("animal")

📝 EXAMEN: ¿Cuándo usar Computer Vision vs Custom Vision?

Usa Computer Vision cuando:

  • ✅ Necesitas análisis general de imágenes
  • ✅ Detección de objetos comunes (personas, vehículos, animales)
  • ✅ OCR de documentos estándar
  • ✅ Implementación rápida sin training

Usa Custom Vision cuando:

  • ✅ Objetos específicos de tu industria
  • ✅ Necesitas mayor precisión en tu dominio
  • ✅ Tienes dataset etiquetado disponible
  • ✅ Requerimientos de compliance específicos

MÓDULO 2.2: CUSTOM VISION

Classification vs Object Detection - Decisión Crítica

Aspecto Image Classification Object Detection
Qué hace "¿Qué es esta imagen?" (una respuesta) "¿Qué objetos hay y dónde?" (múltiples objetos con ubicación)
Ejemplo "Esta imagen es un gato" "Hay 2 gatos en coordenadas (x1,y1) y (x2,y2)"
Casos de uso Control de calidad, categorización Conteo de inventario, detección de equipos
Performance Más rápido, menos datos, menor costo Más lento, más datos, mayor costo, más información

🔍 Nota para Novatos: ¿Cuál elegir?

Pregunta simple: ¿Necesitas saber DÓNDE está el objeto en la imagen?

  • NO: Usa Classification
  • SÍ: Usa Object Detection

🔍 Nota para Novatos: Conceptos de Face API

Face Detection vs Recognition vs Verification:

  1. Detection: "¿Hay caras en esta imagen?" - Encuentra ubicación de caras
  2. Recognition: "¿Quién es esta persona?" - Identifica persona específica
  3. Verification: "¿Es esta la misma persona?" - Compara dos caras

Analogías:

  • Detection: "Veo que hay 3 personas en la foto"
  • Recognition: "Veo que Juan está en la foto"
  • Verification: "Confirmo que estas dos fotos son de la misma persona"

🔍 Nota para Novatos: ¿Cómo entiende el lenguaje una máquina?

Analogía simple: Imagina que eres un recepcionista nuevo. Cuando alguien dice "Quiero cancelar mi reserva del viernes", necesitas entender:

  • Intent (Intención): ¿Qué quiere hacer? → Cancelar
  • Entity (Entidad): ¿Qué información específica da? → Reserva, Viernes
  • Context (Contexto): ¿Hay información previa relevante? → Historial de reservas

Conceptos fundamentales:

  • Intent: La acción que el usuario quiere realizar
  • Entity: Información específica dentro del texto
  • Utterance: Ejemplo de cómo un usuario podría expresar algo
  • Confidence Score: Qué tan seguro está el modelo (0.0 a 1.0)

📝 EXAMEN: Proceso de Migración Obligatorio

  4. Retrain: Entrenar en la nueva plataforma
  5. Test: Validar que performance se mantiene
  6. Switch: Cambiar endpoints en aplicaciones

🔍 Nota para Novatos: ¿Qué es Opinion Mining?

Diferencia con Sentiment Analysis básico:

Sentiment básico: "Esta película es terrible pero la música es increíble" → Sentiment: MIXED

Opinion Mining:

  • Aspecto: "película" → Sentiment: NEGATIVO
  • Aspecto: "música" → Sentiment: POSITIVO

Analogía: Es como tener un crítico experto que puede separar diferentes aspectos de una reseña en lugar de dar solo una opinión general.

🔍 Nota para Novatos: ¿Qué es Prompt Engineering?

Analogía simple: Es como dar instrucciones muy específicas a un asistente muy inteligente pero literal.

Mal prompt: "Escribe algo sobre marketing"

Buen prompt: "Actúa como experto en marketing digital. Escribe un plan de contenido de 30 días para una startup de tecnología B2B, incluyendo tipos de contenido, frecuencia y métricas de éxito. Formato: tabla con columnas Día, Tipo de Contenido, Tema, Canal, Métrica."

Conceptos básicos:

  • System message: Define el rol y comportamiento del AI
  • User message: La pregunta o solicitud específica
  • Temperature: Qué tan creativo vs determinístico (0.0 a 1.0)
  • Max tokens: Límite de longitud de respuesta

MÓDULO 4.1: AZURE COGNITIVE SEARCH - FUNDAMENTOS

¿Qué es Knowledge Mining?

Knowledge Mining es el proceso de extraer información valiosa de grandes volúmenes de contenido no estructurado usando IA. Azure Cognitive Search es la plataforma principal para implementar estas soluciones.

Componentes Clave de Azure Cognitive Search:

Componente Función Descripción
Data Sources Origen de datos Blob Storage, SQL Database, Cosmos DB, SharePoint
Indexers Procesamiento Extraen y procesan datos de las fuentes
Skillsets IA aplicada OCR, NER, sentiment analysis, custom skills
Indexes Almacenamiento Estructura optimizada para búsqueda
Search Service API Expone funcionalidad de búsqueda

🔍 Nota para Novatos: ¿Diferencia entre Search y Database?

Database Query:

SELECT * FROM products WHERE category = 'electronics'
  • Búsqueda exacta
  • Estructura fija
  • Resultados determinísticos

Search Query:

"smartphone with good camera under $500"
  • Búsqueda semántica
  • Contenido flexible
  • Resultados rankeados por relevancia

Analogía simple: Una base de datos es como un archivero organizado. Un search engine es como un bibliotecario experto que entiende lo que realmente necesitas.

Pipeline Típico:

Data Sources → Indexer → Skillset → Index → Search API → Application
     ↓            ↓         ↓        ↓         ↓           ↓
  Documents   Extraction  AI Magic  Storage  Queries   User Interface

MÓDULO 4.2: SKILLSETS Y AI ENRICHMENT

Categorías de Built-in Skills:

Categoría Skills Casos de Uso

Image Skills OCR, Image Analysis, Face Detection Procesamiento de documentos con imágenes

🔍 Nota para Novatos: ¿Qué es el "context" en skills?

El context define el scope donde se aplica el skill:

  • /document: Una vez por documento
  • /document/pages/*: Una vez por página
  • /document/normalized_images/*: Una vez por imagen

Analogía: Es como decir "aplica esta regla a cada párrafo" vs "aplica esta regla a todo el documento".

MÓDULO 4.3: CUSTOM SKILLS DEVELOPMENT

Escenarios para Custom Skills:

  • Integración con APIs externas
  • Lógica de negocio específica
  • Modelos de ML personalizados
  • Transformaciones de datos complejas

Arquitectura de Custom Skill:

Cognitive Search → HTTP Request → Azure Function → External API → Response → Enriched Document

Custom skills siempre requieren:

  1. HTTP endpoint que acepta POST
  2. Formato específico de request/response
  3. Timeout handling (máximo 230 segundos)
  4. Error handling robusto

MÓDULO 4.4: SEARCH INDEXES Y QUERYING

🔍 Nota para Novatos: ¿Qué significan los atributos?

  • searchable: ¿Se puede buscar texto en este campo?
  • filterable: ¿Se puede usar en filtros ($filter)?
  • sortable: ¿Se puede ordenar por este campo?
  • facetable: ¿Se puede usar para faceted navigation?
  • retrievable: ¿Se devuelve en los resultados?

Regla de oro: Solo habilita lo que necesitas. Cada atributo consume storage y afecta performance.

Query Types:

Tipo Descripción Caso de Uso
Simple Query Búsqueda básica con términos Búsquedas de usuarios finales
Full Lucene Query Sintaxis avanzada con operadores Búsquedas complejas y filtros
Semantic Search Comprensión semántica con IA Búsquedas en lenguaje natural

🔍 Nota para Novatos: ¿Qué es un Bot Conversacional?

Diferencia clave:

  • Chatbot simple: Respuestas predefinidas, reglas fijas
  • Bot conversacional: Entiende contexto, mantiene estado, usa IA

Analogía simple: Un chatbot es como un menú de restaurante automatizado. Un bot conversacional es como un mesero experto que recuerda tus preferencias y puede improvisar.

Conceptos básicos:

  • Turn: Una interacción completa (usuario envía mensaje, bot responde)
  • Activity: Cualquier comunicación entre usuario y bot
  • Context: Información que el bot recuerda sobre la conversación
  • State: Datos persistentes del usuario y conversación

Bot Architecture Patterns

Nivel Tipo Características Ideal para
Level 1 Echo/FAQ Bot Respuestas simples, sin estado persistente FAQs, información básica
Level 2 LUIS-Enabled Bot Entiende intenciones, extrae entidades Reservas, pedidos simples
Level 3 Multi-Turn Bot Mantiene contexto, conversaciones complejas Soporte técnico, ventas
Level 4 Enterprise Bot Integra múltiples servicios, autenticación Asistentes corporativos

MÓDULO 5.2: DESARROLLO DE BOTS CON SDK

Activity Types Importantes:

  • Message: Texto del usuario
  • ConversationUpdate: Usuario se une/sale
  • ContactRelationUpdate: Usuario agrega/remueve bot
  • Typing: Usuario está escribiendo
  • DeleteUserData: Solicitud de eliminar datos

🔍 Nota para Novatos: ¿Qué es un Activity Handler?

Activity Handler es como un "despachador de eventos":

  • Recibe diferentes tipos de actividades
  • Decide qué método ejecutar
  • Mantiene el flujo de conversación

Analogía: Es como un recepcionista que dirige visitantes al departamento correcto según su necesidad.

MÓDULO 5.3: DIALOGS Y CONVERSATION FLOW

Tipos de Dialogs:

Tipo Función Caso de Uso
ComponentDialog Dialog base reutilizable Componentes modulares
WaterfallDialog Secuencia de pasos Flujos lineales
PromptDialog Solicita input específico Recolección de datos
ChoicePrompt Opciones múltiples Menús de selección

🔍 Nota para Novatos: ¿Cómo funcionan los Waterfall Dialogs?

Waterfall Dialog es como una cascada de preguntas:

  1. Pregunta 1 → Respuesta → Guarda respuesta
  2. Pregunta 2 → Respuesta → Guarda respuesta
  3. Pregunta 3 → Respuesta → Procesa todo

Analogía: Es como llenar un formulario paso a paso, donde cada pregunta depende de la anterior.

Ventajas:

  • Conversación estructurada
  • Validación en cada paso
  • Puede saltar pasos si ya tiene información
  • Fácil de mantener y modificar

MÓDULO 5.4: CHANNELS Y DEPLOYMENT

Channels Soportados:

Categoría Channels Características
Microsoft Channels Teams, Skype, Cortana, Office 365 Email Integración nativa con ecosistema Microsoft
Third-Party Channels Slack, Facebook Messenger, Telegram, Twilio SMS Alcance amplio de usuarios
Web Channels Web Chat, Direct Line Speech Integración en sitios web y aplicaciones

🔍 Nota para Novatos: ¿Qué es Direct Line?

Direct Line es como un "traductor universal" para tu bot:

  • Tu aplicación web/móvil habla con Direct Line
  • Direct Line habla con tu bot
  • Tu bot responde a través de Direct Line
  • Direct Line traduce la respuesta para tu aplicación

Beneficios:

  • Una sola implementación de bot
  • Funciona en múltiples plataformas
  • Control total sobre la UI

MÓDULO 5.5: BOT FRAMEWORK COMPOSER

Ventajas de Composer:

  • ✅ No-code/Low-code: Diseño visual de conversaciones
  • ✅ Rapid prototyping: Iteración rápida
  • ✅ Integration: Conecta fácilmente con Azure services
  • ✅ Testing: Emulator integrado
  • ✅ Deployment: Deploy directo a Azure

🔍 Nota para Novatos: ¿Cuándo usar Composer vs SDK?

Usa Composer cuando:

  • Prototipado rápido
  • Equipos no técnicos
  • Conversaciones simples a medianas
  • Integración básica con servicios

Usa SDK cuando:

  • Lógica de negocio compleja
  • Integración profunda con sistemas
  • Control total sobre el código
  • Performance crítica

DOMINIO 1: PLANIFICACIÓN Y GESTIÓN (Preguntas 1-8)

Pregunta 1

Una startup de e-commerce procesa 100,000 reviews de productos diariamente en múltiples idiomas. Necesitan análisis de sentimientos en tiempo real con presupuesto limitado. ¿Cuál es la solución más cost-effective?

  1. Azure OpenAI Service con GPT-4
  2. Custom Machine Learning model en Azure ML
  3. Azure AI Language Service con Standard tier
  4. Multiple Cognitive Services con Free tier

Explicación Detallada:

🔍 Análisis Arquitectónico: Esta pregunta evalúa tu capacidad para hacer trade-offs cost-performance apropiados.

Cálculo de Volumen:

  • 100K reviews/día × 30 días = 3M transacciones/mes
  • Standard tier: $2 por 1K transacciones = $6,000/mes
  • Commitment tier S1: $250 base + $2/1K adicional = $4,250/mes (mejor opción real)

Por qué C es correcta:

  • Cost-effective: Precio razonable para el volumen
  • Multiidioma nativo: No requiere traducción previa
  • Tiempo real: Latencia < 1 segundo
  • Escalabilidad: Maneja picos de tráfico

Por qué las otras fallan:

  • A) Azure OpenAI: $0.002/1K tokens × estimado 50 tokens/review = $300/mes en tokens, pero el costo real incluye infrastructure overhead (~$15K/mes total)
  • B) Custom ML: Requiere 3-6 meses desarrollo + $10K/mes infrastructure + expertise ML
  • D) Free tier: 5K transacciones/mes vs 3M necesarias = 99.8% de requests fallarían
Pregunta 2

Su empresa requiere análisis de imágenes médicas para detectar fracturas en rayos X. ¿Qué consideraciones de compliance son críticas?

  1. Solo usar Computer Vision API estándar
  2. Implementar private endpoints y HIPAA compliance
  3. Usar Free tier para reducir costos
  4. Almacenar todas las imágenes en Blob Storage público

Explicación Detallada:

🔍 Análisis de Compliance: Esta pregunta evalúa tu entendimiento de regulatory requirements en healthcare.

Elementos críticos para HIPAA:

  • Private endpoints: Tráfico nunca sale de tu red privada
  • Encryption at rest: Customer-managed keys recomendadas
  • Audit logging: Cada acceso debe ser traceable
  • Access controls: Role-based access con MFA
  • Business Associate Agreement (BAA): Microsoft debe firmar BAA para HIPAA

Por qué las otras fallan:

  • A) Computer Vision estándar: No está entrenado para diagnósticos médicos específicos
  • C) Free tier: Inadecuado para datos de producción sensibles
  • D) Blob público: Violación masiva de HIPAA y privacidad
Pregunta 3

Está diseñando una solución de IA para una empresa financiera. ¿Cuál es la mejor práctica para authentication?

  1. Usar API keys compartidas entre aplicaciones
  2. Implementar Managed Identity con Azure AD
  3. Hardcodear credentials en el código
  4. Usar connection strings en archivos de configuración

Explicación Detallada:

🔍 Análisis de Seguridad Empresarial: Microsoft evalúa tu conocimiento de security best practices para sectores regulados.

Ventajas de Managed Identity:

  • Zero secrets in code: Elimina riesgo de credential exposure
  • Automatic rotation: Azure maneja lifecycle de credentials
  • Audit trail: Integración completa con Azure AD logs
  • Conditional access: Puede aplicar políticas de acceso
  • MFA integration: Hereda políticas de MFA corporativas

🦏 Rhino Tip: Para empresas financieras, Managed Identity + Azure AD es siempre la respuesta correcta sobre API keys. ¡Seguridad con la fuerza de un rinoceronte!

Necesita extraer texto de facturas escaneadas con layouts complejos. ¿Qué servicio es más apropiado?

  1. Computer Vision OCR
  2. Form Recognizer con custom model
  3. Custom Vision con classification
  4. Azure OpenAI con vision capabilities

Explicación Detallada:

🔍 Análisis de Document Processing: Esta pregunta evalúa tu conocimiento de servicios especializados para documentos estructurados.

Por qué B es correcta:

  • Layout understanding: Form Recognizer entiende la estructura de documentos
  • Key-value extraction: Extrae pares clave-valor automáticamente
  • Custom training: Se puede entrenar con tus formatos específicos

Por qué las otras fallan:

  • A) Computer Vision OCR: Solo extrae texto, no entiende estructura
  • C) Custom Vision: Para clasificación de imágenes, no extracción de texto
  • D) Azure OpenAI: Costoso y no optimizado para documentos estructurados

DOMINIO 3: PROCESAMIENTO DE LENGUAJE NATURAL (Preguntas 21-36)

Pregunta 21

¿Cuál es la principal diferencia entre LUIS y Azure AI Language Service?

  1. LUIS es más nuevo que Language Service
  2. Language Service incluye LUIS functionality plus additional capabilities
  3. Son servicios completamente diferentes sin relación
  4. LUIS solo funciona en inglés

Explicación Detallada:

🔍 Análisis de Evolución de Servicios: Esta pregunta evalúa tu conocimiento de la migración de servicios de Microsoft.

Por qué B es correcta:

  • Unified service: Language Service consolida múltiples capacidades
  • LUIS migration: LUIS se está migrando a Conversational Language Understanding (CLU)
  • Additional features: Incluye Text Analytics, Question Answering, etc.
  • Better orchestration: Puede combinar múltiples modelos en un workflow

Capacidades adicionales en Language Service:

  • Sentiment Analysis con Opinion Mining
  • Named Entity Recognition (NER)
  • PII Detection y Redaction
  • Question Answering (ex-QnA Maker)
